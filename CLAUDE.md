## Project Overview

BlueKing Lite documentation website built with Docusaurus 3.8.1. Features custom React components with animations, a playground page with AI demos, pricing page, and multi-language support (Chinese default, English).

## Development Commands

```bash
# Install dependencies
pnpm install

# Start development server (port 3001)
pnpm start

# Start with English locale
pnpm start:en

# Production build
pnpm build

# Clear Docusaurus cache
pnpm clear

# Serve production build locally
pnpm serve

# Docker build and run
docker build -t bklite-website . && docker run -p 80:80 bklite-website
```

## Code Conventions

### Import Order (Strict)

```javascript
// 1. React core
import React, { useState, useEffect } from 'react';

// 2. External libraries (alphabetical)
import clsx from 'clsx';
import { motion } from 'framer-motion';

// 3. Docusaurus/theme imports
import Layout from '@theme/Layout';
import Link from '@docusaurus/Link';
import useDocusaurusContext from '@docusaurus/useDocusaurusContext';

// 4. Internal components (@site/*)
import HomepageFeatures from '@site/src/components/HomepageFeatures';

// 5. Styles (ALWAYS LAST)
import styles from './index.module.css';
```

### Component Patterns

- **JavaScript only** - no TypeScript
- **Functional components only** - no class components
- **PascalCase** for component names and folders
- **camelCase** for functions and variables
- Use CSS Modules: `import styles from './styles.module.css'`
- Use `clsx` for conditional classes
- Default exports for main page components

### Styling

- Use Infima CSS variables: `var(--ifm-color-primary)`, `var(--ifm-spacing-horizontal)`
- Global fonts are CJK-optimized in `src/css/custom.css` - do not modify `!important` rules
- Animation components available in `src/components/ScrollAnimations/`

### Animation Pattern

```javascript
import { FadeInUp, StaggerContainer, StaggerItem } from '@site/src/components/ScrollAnimations';

<FadeInUp delay={0.1}>
  <div>Content fades in on scroll</div>
</FadeInUp>

<StaggerContainer staggerDelay={0.08}>
  {items.map((item, idx) => (
    <StaggerItem key={idx} direction="up">
      <Card {...item} />
    </StaggerItem>
  ))}
</StaggerContainer>
```

## Key Configuration Files

- `docusaurus.config.js` - Site config, navbar, footer, i18n (zh-Hans default), analytics
- `sidebars.js` - Documentation sidebar structure (autogenerated per product)
- `nginx.conf` - Nginx configuration for Docker deployment

## Pre-commit Hook

Changes to `deploy/docker-compose/` or `deploy/dev/` automatically regenerate install scripts:

```bash
# Triggered by .husky/pre-commit
# Regenerates: static/install.run, static/install.dev
# Syncs: deploy/dev/conf/ from docker-compose
```

## i18n

- Default locale: `zh-Hans` (Chinese Simplified)
- Secondary locale: `en` (English)
- Translations in `i18n/` directory
- Use `<Translate>` component or `translate()` function for user-facing strings

## Custom Fields

Available via `siteConfig.customFields`:
- `apiBaseUrl` - API endpoint for playground
- `loginBaseUrl` - Authentication endpoint
- `loginInfoUrl` - User info endpoint

## Testing

No test framework configured. Always verify with `pnpm build` before committing.

## Deployment

```bash
# Build static site
pnpm build

# Docker deployment
docker build -t bklite-website .
docker run -p 80:80 bklite-website
```

## Important Notes

- Do not use relative imports - use `@site/` or `@theme/` aliases
- No Redux or external state management - use React hooks only
- Chinese comments are acceptable and match the default locale
- The playground page requires authentication integration (third_login_code handling)
