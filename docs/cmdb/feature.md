# 功能介绍

## 一、模型管理

模型管理是 BK-Lite CMDB 的基础能力，用于定义“系统中有哪些资产、每类资产有哪些字段、如何分组展示、如何与其他对象建立关系”。一旦模型建立完成，后续实例录入、导入导出、搜索、详情页展示和自动发现写入，都会围绕该模型规则执行。

> **界面指引：**
>
> ![模型管理](https://static.cwoa.net/3bb26a49a5474c32a19ef44b252388b7.png)
>
> *   **图表解读/配置逻辑**：模型管理页既是资产标准入口，也是后续实例维护的规则源头。分类决定资产的组织方式，模型决定资产的数据结构，关系决定资产在拓扑中的连接逻辑。

### 1. 模型分类与模型定义

*   **分类化组织**：支持按业务域、资源类型或管理边界建立模型分类，方便大规模资产的结构化管理。
*   **模型标准化定义**：支持新增、编辑、删除模型，并以模型为单位统一维护属性、关系和展示规则。
*   **模型复制复用**：支持复制已有模型，并按需复制属性、字段分组和关系配置，适合快速扩展标准模型体系。
*   **组织范围控制**：模型可结合组织维度设置适用范围，为多团队共享同一套 CMDB 提供边界管理基础。

### 2. 属性管理

*   **丰富属性类型**：支持字符串、整数、枚举、时间、用户、组织、布尔、密码、标签、表格等类型，覆盖常见资产信息建模场景。
*   **约束规则统一**：支持必填、唯一、可编辑等字段约束，保证实例录入和维护过程中数据规则一致。
*   **默认字段内置**：模型默认包含实例名和所属组织等关键基础字段，减少重复配置。
*   **显示字段补齐**：平台会为部分目标类型自动补充显示字段定义，帮助实例详情和列表页更直观地展示用户、组织等对象信息。

### 3. 字段分组与展示结构

*   **字段分组管理**：支持创建、编辑、删除字段分组，并控制分组顺序。
*   **页面结构优化**：支持配置分组折叠状态，让详情页和编辑页更适合复杂模型的阅读与维护。
*   **录入体验更清晰**：通过分组组织字段，用户在录入或查看复杂资产时无需面对冗长平铺表单。

### 4. 模型关系管理

*   **关系类型标准化**：支持“属于、组成、运行于、安装于、包含、关联”等多种关系类型。
*   **映射约束控制**：支持一对一、一对多、多对一、多对多等关系约束，避免随意关联导致结构失真。
*   **双向结构表达**：模型关系不仅服务于实例关联，也直接影响详情页关系展示与拓扑视图的表达能力。
*   **图关系存储基础**：模型之间的关系以图结构方式表达，更适合复杂依赖和拓扑分析场景。

## 二、资产管理

资产管理围绕实例的全生命周期展开，覆盖从创建、修改、批量导入导出，到关系维护、详情查看和变更追踪的完整操作链路。

### 1. 实例新增与维护

*   **单条实例新增**：按模型定义录入实例属性，适合精细维护关键资产。
*   **实例编辑与删除**：支持持续维护可编辑字段，保证资产数据随业务变化同步更新。
*   **批量操作能力**：支持批量导入、批量删除、批量导出，适合初始化纳管和大批量数据维护。
*   **组织归属统一**：实例默认带有组织归属字段，便于结合权限策略进行统一管理。

> **界面指引：**
>
> ![资产列表](https://static.cwoa.net/6c15745c0d35475ea243a7cb3561ec1a.png)
>
> *   **图表解读/配置逻辑**：资产列表页既是日常维护入口，也是批量操作中心。用户可以基于模型快速录入实例，也可以结合筛选条件、导入导出和批量处理能力完成规模化维护。

### 2. 导入与导出

*   **Excel 导入**：支持通过模板批量导入实例，适合历史资产迁移与初始盘点。
*   **按模型导出**：支持导出当前模型下的实例数据，用于离线分析、交接和汇报。
*   **按字段控制导出范围**：可按用户选择导出字段，减少无关信息干扰。
*   **关联信息导出**：支持把实例关联关系一并导出，帮助在表格层面保留结构信息，而不是只导出孤立属性。

### 3. 关联关系维护

*   **按模型关系创建关联**：实例关联建立在模型关系规则之上，减少错误连接。
*   **支持取消关联**：当依赖关系不再成立时，可手动解除。
*   **按关系类型查看**：实例详情页支持按关系类型分类展示关联对象。
*   **关系与实例联动查看**：点击关联对象可继续跳转查看对方详情，形成问题追踪链路。

### 4. 实例详情与拓扑

*   **基础信息集中查看**：详情页集中展示实例核心属性，减少来回切换。
*   **关系列表与拓扑双视角**：既可按清单方式查看关联，也可按拓扑方式理解依赖层级。
*   **按层展开与节点扩展**：拓扑视图支持逐步展开，适合复杂结构场景下的影响分析。
*   **结构化排障入口**：从单个实例进入上下游依赖关系，是故障定位和资源梳理的重要入口。

> **界面指引：**
>
> ![资产关系拓扑](https://static.cwoa.net/0ad2e33d02da46659f5c61e1835e694c.png)
>
> *   **图表解读/配置逻辑**：拓扑页将实例关系从“表格式记录”转化为“结构化依赖视图”。用户可以围绕一个对象持续扩展关联节点，更快识别上下游影响范围。

## 三、资产视图与全文检索

资产数据规模扩大后，真正影响使用体验的不是“能不能存”，而是“能不能快速找到”。BK-Lite CMDB 通过资产视图和全文检索两条入口，分别解决全局浏览与精准定位的问题。

### 1. 资产视图

*   **按分类汇总展示**：平台按模型分类聚合资产，帮助用户快速理解资产分布。
*   **模型级实例统计**：在总览页直接展示模型实例数量，建立全局认知。
*   **关键字筛选**：支持按模型名称搜索，快速收敛浏览范围。
*   **一键跳转实例列表**：从总览卡片直接进入对应模型的资产列表，提高操作连续性。

### 2. 全文检索

*   **跨模型搜索**：输入关键字后，可在多模型范围内快速查找资产实例。
*   **按模型统计结果**：搜索结果会先给出模型维度统计，再支持继续下钻。
*   **分页查看与详情预览**：支持在搜索结果中直接查看实例详情并定位目标对象。
*   **大小写敏感控制**：适合需要精确匹配标识、主机名、实例名等场景。
*   **本地搜索历史保存**：保留常用检索关键字，方便高频使用者快速回查。

> **界面指引：**
>
> ![全文检索](https://static.cwoa.net/939e5dc090204c37b5c739a9635f6ba1.png)
>
> *   **图表解读/配置逻辑**：全文检索不是简单返回一串命中记录，而是先按模型汇总结果，再支持查看每个模型下的实例详情。这种方式更适合企业环境中多模型并存的真实查找习惯。

## 四、自动发现与采集任务

自动发现是 BK-Lite CMDB 保持资产数据新鲜度的重要手段，适合处理数量大、变化快、手工维护成本高的对象。平台以采集对象树组织发现范围，并围绕任务配置、执行、摘要、审批与结果查看形成闭环。

### 1. 采集对象与适用范围

*   **多对象纳管**：支持 K8s、VMware、网络设备、数据库、云平台、主机、中间件等类型。
*   **对象树式入口**：按采集对象组织任务入口，降低任务配置复杂度。
*   **插件能力承载**：不同对象类型匹配不同采集逻辑和说明文档，便于标准化落地。
*   **协议与脚本双驱动**：既支持协议采集，也支持脚本方式扩展覆盖面。

### 2. 采集任务配置

*   **周期执行配置**：支持开启周期巡检并设置执行周期。
*   **接入点与实例范围控制**：支持配置接入点、IP 范围或具体实例范围。
*   **凭据与超时控制**：支持配置账户、密码、端口及单对象超时时间，满足不同采集环境要求。
*   **参数扩展能力**：支持为不同对象类型附加额外参数，增强采集灵活性。
*   **组织关联管理**：任务可绑定组织范围，便于权限控制和团队协同。

### 3. 执行结果与审批写入

*   **结果摘要分类**：任务执行后自动统计新增、更新、删除、关联及异常数量。
*   **采集明细可追溯**：可查看格式化后的结果数据与原始采集数据摘要。
*   **审批后写入**：对需要人工确认的任务，可先进入待审批状态，再决定是否写入资产台账。
*   **异常状态兜底**：平台会对长时间运行未完成的任务进行状态回收，避免任务状态长期悬挂。

### 4. 安全与准确性保障

*   **凭据加密存储**：任务凭据在保存时会对敏感字段执行加密处理。
*   **接口返回统一脱敏**：密码类字段在返回前统一替换为掩码，减少泄露风险。
*   **按采集模型识别加密字段**：不同采集对象可按模型定义敏感字段，兼顾扩展性与安全性。
*   **结果摘要辅助复核**：通过分类统计快速判断本次采集是否符合预期，避免盲目入库。

> **界面指引：**
>
> ![采集任务详情](https://static.cwoa.net/3037f896fd0e4d99b21ee9de1cdc95eb.png)
>
> *   **图表解读/配置逻辑**：任务详情页不是单纯展示“任务成功/失败”，而是把新增、更新、删除、关联等结果分开呈现，帮助管理员快速判断这次任务究竟改变了什么。

## 五、场景化发现能力

在自动发现之上，BK-Lite CMDB 针对部分典型场景具备更深入的结构恢复能力，不只是采集“对象清单”，而是尽量恢复“对象关系”。

### 1. K8s 资源关系恢复

*   **分层识别 K8s 资源**：支持集群、命名空间、工作负载、Pod、节点等对象纳管。
*   **自动补齐层级关系**：可恢复命名空间属于集群、Pod 运行于节点、Pod 隶属于工作负载等关系。
*   **工作负载结构还原**：在部分场景下可通过 ReplicaSet 等信息追溯到更上层工作负载。
*   **更适合容器环境拓扑展示**：帮助团队理解容器资源的运行结构，而不是只拿到散点数据。

### 2. 网络设备与接口拓扑

*   **设备识别依赖 SOID 特征库**：通过 sysObjectID 映射设备型号、品牌和类型，提升识别准确率。
*   **接口级数据采集**：可采集接口信息，并补齐接口对象。
*   **接口连接关系推导**：结合 ARP、MAC、接口信息推断接口之间的连接关系。
*   **网络拓扑更接近真实结构**：适合交换机、网络设备等场景下做连接关系梳理。

### 3. 主机与其他基础设施对象

*   **支持主机发现与周期巡检**：适合做存量纳管和日常补采。
*   **可配置实例或 IP 范围**：兼顾精确补采与批量发现。
*   **适用 VMware、数据库、中间件等对象**：通过插件和对象树扩展多种基础设施纳管范围。

## 六、公共选项库与个人配置

### 1. 公共选项库

*   **统一维护枚举选项**：适用于状态、等级、分类等跨模型复用的字段。
*   **模型属性直接引用**：枚举属性可直接引用公共选项库，避免在多个模型中重复配置。
*   **引用关系校验**：删除选项库前可检查引用情况，避免误删影响已使用模型。
*   **变更同步刷新**：当选项库更新后，可同步刷新引用该库的模型属性快照，保持配置一致。

### 2. 个人配置

*   **展示字段配置**：用户可按模型保存更适合自己职责的列表展示字段。
*   **常用筛选条件保存**：支持保存常用过滤条件，提高高频查询效率。
*   **贴合角色工作方式**：不同岗位可基于同一份资产数据形成不同查看视角，而无需改变底层模型设计。

## 七、权限规则与变更记录

### 1. 权限规则

*   **组织级可见范围控制**：可按组织范围控制模型、实例和采集任务的可见性。
*   **对象级操作边界**：不同用户可拥有不同的新增、编辑、删除、管理权限。
*   **默认组织兜底**：默认组织下的模型可作为基础共享范围，便于平台初始化落地。
*   **多团队协作更安全**：在同一套平台上支持不同团队共享资产体系而不过度暴露权限。

### 2. 变更记录

*   **全程留痕**：记录实例修改、关系调整、采集写入等关键操作。
*   **按时间范围筛选**：便于围绕变更窗口排查问题。
*   **详情可追溯**：支持查看变更详情，帮助复盘“谁在什么时候改了什么”。
*   **审计与排障兼顾**：既可服务合规审计，也可服务故障回溯与问题定位。

> **注意 / 安全最佳实践：**
> 对于自动发现任务，建议使用最小权限凭据，并在正式批量执行前先用小范围目标进行验证。这样既能降低凭据风险，也能减少误采集或错误写入对资产台账造成的影响。
